Connection Lifecycle OS
The Single Source of Truth
Defines what a connection is, what states it can be in, and what moves it between them. It is a state machine — it does not decide, judge, or punish.
6 Connection States
REQUESTED → PENDING INSTALL → ACTIVE ↔ PAUSED → PENDING DEACTIVATION → DEACTIVATED
Only ACTIVE connections count
  • #Counted in stock (Capacity OS uses this for partner caps)
  • QMeasured by Quality OS (uptime, resolution, install quality)
  • $Eligible for recharges that drive compensation earnings

  • ⊕Recharge = resume. No recharge = PAUSED. Recharge confirmed = back to ACTIVE. No auto-resume.
  • ∅DEACTIVATED = terminal. No coming back. Connection ID never reused.
  • ≅Reassignment doesn't change state. If D&A reassigns the partner, an ACTIVE connection stays ACTIVE.

No retroactive changes — ACTIVE at time T stays counted as ACTIVE for that period
Card 2 of 4
Quality OS
The Truth Machine
Measures every partner across 3 independent domains and declares their SLA state. It only observes — it never punishes directly.
3 Domains (evaluated independently)
  • SService Stability — Is the network up? (uptime %)
  • RService Resolution — Are tickets resolved on time? (breach count)
  • IInstallation Quality — Are installs done right? (rework rate %)
Partner SLA States
COMPLIANT → AT RISK → NON-COMPLIANT → RECOVERY
  • !See it first. Partner sees their state on dashboard BEFORE any consequence fires — self-correction window (P15)
  • TNo hiding in the middle. Stuck AT RISK beyond P17 windows? Auto-escalated to NON-COMPLIANT
  • ✓COMPLIANT = full reset. Everything clears — counters, flags, all downstream consequences reset
  • NDomains never blend. A partner can be Compliant in one domain and Non-Compliant in another
What it triggers
AT RISK → Enforcement enters watch mode, bonus flagged for review
NON-COMPLIANT → Routing tapered, bonus paused, enforcement evaluates restriction
Sustained NC → Feeds viability signal to Exit OS

Enforcement OS
The Consequence Engine
Converts Quality's truth into participation consequences. Two parallel tracks that never merge — performance (gradual) and integrity (immediate).
Two Parallel Tracks  — tap a track to flip
Performance Track
Gradual escalation
CLEAR → WATCH → RESTRICT → PRE-EXIT
tap to see rules  ↻
Performance Rules
  • Counts consecutive NC windows — E02 to restrict, E03 more to pre-exit
  • COMPLIANT = instant full reset — no probation period
  • Only system escalates — no manual override of track state
tap to flip back  ↻
Integrity / FPV Track
Instant consequence
FLAGGED → SUSPENDED → TERMINATED
tap to see rules  ↻
Integrity Rules
  • Instant — fraud, disintermediation, safety bypass all waiting
  • TERMINATED = permanent. Irreversible. No coming back.
  • Suspended = zero routing, all payouts frozen immediately
tap to flip back  ↻
Key Rules
  • ~Performance = gradual. Counts consecutive NC windows. E02 to restrict, E03 more to pre-exit.
  • ≠Integrity = instant. Fraud, disintermediation, safety — bypasses all waiting.
  • ✓COMPLIANT = instant full reset. No probation. Performance track clears completely.
  • ×TERMINATED = permanent. Critical integrity violation = irreversible exit.
  • =Both active? More severe posture wins. No double-stacking of consequences.
Consequences at each stage
RESTRICTED → Routing tapered, bonus paused, CSP notified
PRE-EXIT → Exit OS starts viability assessment (B1)
SUSPENDED → Zero routing, all payouts frozen
TERMINATED → Permanent exit (B2), irreversible
Card 4 of 4
Demand & Allocation OS
The Router
Decides who serves every connection. Fully deterministic, no manual override, no territory entitlement. Better quality = more connections.
Routing Priority (strict order)
1
Eligible? — Not exited, not suspended, within capacity cap (P46)
2
Quality band — Compliant beats At Risk. At Risk beats Baseline. Always.
3
Utilization — Same quality? Less loaded partner gets priority.
4
Tiebreaker — Earlier onboarding date wins. Never random.
Exposure Bands
FULL → REDUCED (0.5x) → BASELINE (0.3x) → INELIGIBLE
  • ≠No territory rights. Partners earn connections through quality, not location ownership.
  • ∅No eligible partner? Clean failure. System does NOT force-assign or work around it.
  • %Zone cap at 60%. If one partner holds >60% of a zone, growth is throttled (P56).
  • ≅Sticky post-accept. Only system triggers (exit, enforcement, quality) can reassign.
The core incentive loop
Stay compliant = get more connections = earn more
Drop in quality = fewer connections automatically
Game the system = integrity track = zero connections
