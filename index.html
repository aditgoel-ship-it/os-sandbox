🧠 Wiom OS Guide

An AI-powered Slack assistant trained on Wiom's entire CSP Operating System architecture — 29 documents covering 20 OS across vision, primitives, governance, and execution.

What it does:

  • Answers any CSP operations question — enforcement, compensation, exits, quality, routing, asset custody, payments, capacity — all grounded in locked OS docs
  • Derives answers even for undrafted OS — uses primitives, guards, and architecture docs to give principled guidance when a formal OS doesn't exist yet
  • Auto-flags edge cases to SD — when it detects a gap or high-stakes scenario, it flags automatically to #os-feedback
  • Reply "details" to any answer to get the full mechanics (state IDs, invariants, section references)

How to use: DM the Wiom OS Guide app directly on Slack and ask your question. You can also mention @Wiom OS Guide in any channel. Type flag to escalate the last answer to SD for review.

How We Work

  • System design drives workflow — not the other way around. The backend follows the OS logic, not ad-hoc corrections.
  • Standups = alignment only. No status recitation. If you're blocked, say it. If not, move on.
  • Small doubts = direct meeting with the relevant person. Don't wait for standup.
  • Design = agent-first. Build everything through AI agents. Manual intervention only when truly not possible.
  • Anyone can submit an amendment — but ask the GPT first to understand implications.
